2. **Polymorphic and Metamorphic Malware**:
   - **Polymorphic Malware**: Changes its code slightly each time it infects a new system, typically using an encryption algorithm to modify its appearance while keeping its functionality intact.
   - **Metamorphic Malware**: Goes further by completely rewriting its code with each infection, making detection through static signatures extremely difficult.

3. **Fileless Malware**:
   - Operates directly in memory rather than installing files on disk, thereby evading detection by file-based antivirus scans. It often leverages legitimate system tools like PowerShell or Windows Management Instrumentation (WMI).

4. **Packers and Crypters**:
   - Attackers use these tools to compress and encrypt malware payloads. The resulting packed files appear benign and are frequently repacked, rendering static signature detection ineffective.

5. **Living off the Land (LotL) Attacks**:
   - Utilize legitimate system tools and processes to carry out malicious activities, making them appear as normal system operations, which can evade detection by antivirus solutions.

6. **Exploiting Zero-Day Vulnerabilities**:
   - By leveraging vulnerabilities that are unknown to the vendor and for which no patch is available, attackers can evade antivirus software that relies on signature updates.

7. **Signature Evasion**:
   - Creating slight variations of known malware or using techniques to dynamically change the signature, making it difficult for traditional signature-based detection methods to identify the threat.

8. **Behavioral Evasion**:
   - Designing malware to delay malicious behavior until specific conditions are met, thereby avoiding detection by behavior-based detection during a sandbox analysis phase.

9. **Environment-Aware Malware**:
   - Malware that checks for sandbox or virtual machine environments and alters its behavior or stays dormant to avoid being detected during dynamic analysis.

10. **DNS and Domain Generation Algorithms (DGAs)**:
    - Use of unpredictable domain names for command-and-control (C2) communications is challenging for antivirus systems that rely on blocking known malicious domains.

11. **Code Injection and Process Hollowing**:

12. **Evasion through User Interaction**:
