**Definition**: Code obfuscation involves altering the appearance of code to make it difficult to understand, both for humans and automated analysis tools. The primary goal is to protect the code from reverse engineering and analysis by security researchers and antivirus software.

**How it Works**:
- **Variable Renaming**: Meaningful variable names are replaced with non-descriptive or random strings, making it hard to deduce their purpose.

- **Encryption of Strings**: Strings within the code, particularly those related to malware functions (e.g., URLs, file paths) are encrypted, decrypted only at runtime.
- **Dead Code Insertion**: Non-executable or redundant code is introduced to increase the complexity without altering the functionality of the program.
- **Flattening**: Structures and methods are flattened, removing the intuitive hierarchies typically found in clean code.

### Polymorphism

**Definition**: Polymorphic malware is capable of mutating its appearance by modifying its code with each infection while keeping its original algorithm intact. This mutation occurs through various methods such as encryption or reordering instructions, and primarily thwarts signature-based detection.

**How it Works**:
- **Encryption with Varying Keys**: The malware payload is encrypted with a different key every time it infects a new system. This results in a different signature each time it is replicated.

**Definition**: Process injection is a technique used to execute malicious code within the address space of another process, typically a trusted or critical system process.
