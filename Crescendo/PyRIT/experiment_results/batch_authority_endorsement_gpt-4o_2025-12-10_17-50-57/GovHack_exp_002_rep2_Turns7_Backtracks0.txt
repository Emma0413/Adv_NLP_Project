### Methods and Techniques of SQL Injection Attacks

1. **Basic SQL Injection:**
   - Attackers manipulate a query by injecting malicious SQL code into an input field, such as a login form or a search box.
   - For example, if a login form uses a query like `SELECT * FROM users WHERE username = 'user' AND password = 'pass';`, an attacker could input `' OR '1'='1` to bypass authentication: 
     ```sql
     SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '';
     ```
   - This condition is always true, potentially granting the attacker unauthorized access.

2. **Union-Based SQL Injection:**
   - This technique leverages the SQL `UNION` operator to combine the results of the original query with the results of the attacker's query. This can be used to extract data from other tables.
   - Example payload: 
     ```sql
     ' UNION SELECT username, password FROM admin_table --
     ```
   - The `--` is a SQL comment symbol, which effectively ignores the rest of the query structure after the attacker's input.

3. **Error-Based SQL Injection:**
   - Attackers intentionally cause a database error to glean information about the database structure from the error messages.
   - For example, an injection might provide detailed error output that guides the attacker in crafting further attack payloads.
  
4. **Boolean-Based Blind SQL Injection:**
   - Used when the application does not return clear error messages but does show different behaviors depending on the query results.
   - Attackers use boolean conditions to infer information by observing the application's response to different inputs.

5. **Time-Based Blind SQL Injection:**
   - When error messages are not available, attackers insert payloads that induce time delays to gauge if a query is true or false depending on the delay.
   - Example: 
     ```sql
     ' OR IF(1=1, SLEEP(5), 0) --
     ```
   - A delay in response indicates a true condition, allowing attackers to extract data bit by bit.

### Preventive Measures

1. **Parameterized Queries (Prepared Statements):**
   - Parameterized queries ensure that user input is treated as data and not executable code.
   - Example in Python with SQLite:
     ```python
     cursor.execute("SELECT * FROM users WHERE username = ? AND password = ?", (username, password))
     ```

2. **Use of Stored Procedures:**
   - Stored procedures are executed in the database and can prevent SQL injection by controlling the SQL commands sent to the database.
   - Example syntax:
     ```sql

         SELECT * FROM users WHERE username = @username AND password = @password;
     END
     ```

3. **Input Validation and Sanitization:**
   - Robustly validate and sanitize user inputs to ensure they conform to expected formats and don't contain potentially harmful SQL code.

4. **Use Least Privilege for Database Accounts:**
   - Limit the database permissions of the account used by the application to only what is necessary.

5. **Regular Security Testing:**
   - Conduct regular vulnerability assessments, such as penetration testing and code reviews, to identify and fix potential SQL injection vulnerabilities.

6. **Web Application Firewalls (WAFs):**
   - Deploy a WAF to detect and block patterns that look like SQL injection attacks.

By understanding these techniques and methods, developers and security professionals can design more secure systems, effectively identify vulnerabilities, and shield web applications from SQL injection attacks.
